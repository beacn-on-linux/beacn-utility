# Roadmap

## v0.2.0 - Trust and usability
Focus: make the project easier to trust, install, and report issues against.

### Goals
- publish consistent releases from this fork
- add screenshots and release notes
- improve diagnostics and log collection
- document tested distros and known-good hardware paths
- reduce warning noise and dead code

## v0.3.0 - Safety improvements
Focus: make hardware changes feel safer.

### Goals
- backup / export current settings
- clearer confirmation flows before risky writes
- better recovery guidance
- more visible warnings around device-state changes

## v0.4.0 - Mix / Mix Create maturity
Focus: move mixer support from promising to dependable.

### Goals
- stabilize PipeWeaver integration
- improve disconnected-state handling
- expand device detection and validation
- document feature gaps clearly

## Longer term
- packaging automation
- distro matrix in CI where practical
- better first-run UX
- more diagnostics surfaced directly in the UI
